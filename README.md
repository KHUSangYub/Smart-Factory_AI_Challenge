# Description

🏭 **2026 Edge AI Challenge: 극한의 스마트 팩토리를 구하라!**

🚨 **MISSION: 초고속 컨베이어 벨트를 감시하라**

여러분은 글로벌 철강 기업의 AI 비전(Vision) 엔지니어입니다. 우리 공장의 생산량을 책임지는 핵심 컨베이어 벨트는 생산성 극대화를 위해 **초당 5미터(5m/s)**라는 맹렬한 속도로 돌아가고 있습니다.

여러분의 임무는 이 벨트 위를 지나가는 철강 표면을 실시간으로 촬영하여, **머리카락 굵기의 미세한 긁힘과 균열(불량)**을 완벽하게 찾아내는 AI 모델을 개발하는 것입니다.

하지만 현장의 상황은 여러분이 평소 연구실에서 다루던 환경과 완전히 다릅니다.

---

🚧 **현실의 벽 (Pain Points)**

여러분이 돌파해야 할 1가지 치명적인 제약 조건이 존재합니다.

**1. 하드웨어 한계: 엣지 컴퓨팅 (Edge Computing)**

공장의 보안과 실시간성 확보를 위해, 이미지를 중앙 서버(Cloud)로 전송할 수 없습니다. 여러분의 AI는 카메라에 부착된 손바닥만 한 **저전력 보드(CPU)**에서 즉각적으로 판별을 내려야 합니다. 즉, **무겁고 거대한 모델(ResNet, ViT 등)은 너무 느려서 생산 라인에 병목을 유발하므로 절대 사용 할 수 없습니다.**

🎯 **여러분의 목표 (Your Goal)**

> "가장 무거운 모델이 아니라, 가장 가벼우면서도 예리한 모델을 설계하세요."

주최측은 여러분에게 **매우 선명하고 깨끗하게 찍힌 철강 표면 이미지(Train Data)**를 제공합니다. 하지만 여러분의 모델이 채점되는 평가 서버(Hidden Test Data)에는 **모션 블러가 강하게 낀 악조건의 데이터**가 기다리고 있습니다.

1. 주어진 데이터를 어떻게 훈련시킬 것인가? (Data-centric AI)
2. 무거운 모델의 성능을 가벼운 모델(Student)에 이식할 것인가? (Knowledge Distillation)
3. CPU 환경에서 얼마나 빠르게 모델을 추론시킬 것인가? (Model Optimization)

이 세 가지 질문에 대한 답을 찾아가는 과정이 곧 이 대회의 핵심입니다.

---

💡 **HINT: 살아남기 위한 전략**

얕고 가벼운 모델(MobileNet, ShuffleNet 등)은 속도는 빠르지만, 흐릿한 스크래치를 볼 수 있는 시력이 부족합니다. 반면, 무거운 모델은 블러 속에서도 힌트를 찾아내지만 너무 느립니다.

여러분이 우승하기 위해서는 무거운 스승(Teacher) 모델이 파악한 미세한 특징과 확률 분포를 가벼운 제자(Student) 모델에게 전수하는 **지식 증류(Knowledge Distillation)** 기법을 적극적으로 연구하고 적용해야 합니다. 또한, 깨끗한 Train 데이터에 스스로 노이즈를 섞어보는 **데이터 증강(Data Augmentation)** 기법도 훌륭한 무기가 될 것입니다.

- **최적화**: 추론 시간 패널티를 고려하여 무작정 모델 크기만 키우지 않기
- **양자화**: 모델의 용량과 연산 속도를 효율화하는 기법 적용
- **지식 증류**: 거대 모델의 지식을 작은 모델에 전수하여 효율성 증대
- **프루닝**: 불필요한 가중치를 제거하여 모델을 가볍게 압축

# Evaluation

본 대회는 스마트 팩토리의 엣지 환경을 모사하기 위해, 모델의 정확도(Accuracy)뿐만 아니라 **추론 속도(Inference Time)**를 동시에 평가합니다.

제출된 파일은 **Macro F1-Score**를 기반으로 한 기본 점수에서, 목표 시간을 초과한 만큼의 **시간 패널티(Time Penalty)**를 차감하여 최종 점수(Final Score)를 산출합니다.

**최종 점수 산출 공식 (Custom Metric)**

최종 점수는 다음과 같이 계산됩니다:

$$\text{Final Score} = \max(0, (\text{Macro F1} \times 100) - \max(0, (T_{inference} - 60) \times 0.2))$$

- **Macro F1**: 예측 확률과 실제 타겟 간의 Macro F1-Score (0 ~ 1 사이의 값)
- **$T_{inference}$**: 전체 Test 데이터셋을 CPU로 추론하는 데 걸린 총 소요 시간 (초)
- **목표 시간 (Target Time)**: 60초. (60초 이하로 추론을 완료하면 감점이 없습니다.)
- **감점 비율 (Penalty Weight)**: 60초를 초과한 시점부터 1초당 **0.2점씩** 선형 감점됩니다.
- 🚨 **타임아웃 (Time Out)**: 총 추론 시간이 **300초(5분)**를 초과할 경우, 해당 제출물은 **실격(0점)** 처리됩니다.

# 📊 평가 기준 (Evaluation Criteria)

본 대회의 최종 점수는 모델의 정확도를 나타내는 **성능 점수**와 추론 속도를 나타내는 **속도 점수**를 고려하여 산출합니다.

### 2. 세부 평가 지표

- 🎯 **성능 평가 지표 (Performance Metric):** `Macro F1 Score`
- ⏱️ **속도 평가 지표 (Speed Metric):** `Inference Time` (추론 시간)

### 3. 페널티 (Penalty) 부여 규칙

추론 시간(Inference Time)이 기준 시간을 초과할 경우 감점이 발생합니다.

- **기준 시간 (Target Time):** 60초
- **페널티 계산식:**
  $$    ext{Penalty} = \max(0, (    ext{Inference Time} -     ext{Target Time})     imes 0.2)$$
- 🚨 **실격 조건:** 단, Inference Time이 **300초**를 초과할 경우 **실격 (0점)** 처리합니다.

**Submission File Format**

참가자는 캐글 노트북 환경( `Accelerator: None` )에서 베이스라인 코드를 실행하여 추론 시간을 직접 측정해야 합니다. Test 셋의 각 `Id` (이미지 파일명)에 대해 예측한 클래스 인덱스(0~5)를 `Expected` 열에 기록해야 합니다.

# Dataset Description

본 데이터셋은 스마트 팩토리의 철강 제조 공정에서 수집된 표면 이미지 데이터입니다. 참가자들은 이 데이터를 활용하여 철강 표면의 다양한 결함(Defect)을 분류하는 경량화 AI 모델을 개발해야 합니다.

### 데이터 구성

데이터는 크게 학습용(Train) 데이터와 평가용(Test) 데이터로 나뉘며, 모델의 강건성(Robustness)을 테스트하기 위해 두 세트 간에 중요한 차이가 존재합니다.

- **Train Data (학습 데이터):**
  - 조명이 잘 통제된 실험실 환경에서 고해상도 카메라로 촬영되었습니다.
  - 초점이 정확하게 맞아 결함이 **매우 선명하고 깨끗하게(Clean)** 보입니다.
  - 모델이 각 결함의 특징을 명확하게 학습할 수 있도록 설계되었습니다.
- **Test Data (평가 데이터 - Hidden):**
  - 초당 5m로 돌아가는 실제 현장의 고속 컨베이어 벨트 위에서 촬영되었습니다.
  - 빠른 이동 속도로 인해 이미지에 **강한 모션 블러(Motion Blur)**가 포함되어 있습니다.
  - 조명 변화, 미세한 먼지 등 실제 엣지(Edge) 환경의 노이즈가 섞여 있어, 선명한 데이터로만 학습한 모델에게는 매우 어려운(Hard) 조건입니다.

---

### 클래스 정보 (Class Labels)

데이터는 정상(Normal)을 포함하여 총 **6개의 클래스(0~5)**로 구성되어 있습니다. 각 클래스는 철강 표면에서 흔히 발생하는 고유한 불량 유형을 나타냅니다.

- `0`: **정상 (Normal)** - 결함이 없는 깨끗한 철강 표면
- `1`: **찍힘 (Punch)** - 날카로운 물체에 의해 표면이 국소적으로 움푹 파인 결함
- `2`: **스크래치 (Scratch)** - 표면이 긁혀서 생긴 길고 얇은 선 형태의 결함
- `3`: **균열 (Crack)** - 철강 내부 응력 등에 의해 표면이 미세하게 갈라진 결함 (매우 찾기 어려움)
- `4`: **이물질 (Inclusion)** - 제조 과정 중 쇳물에 이물질이 섞여 들어가 굳어진 결함
- `5`: **오염 (Stain)** - 표면에 기름, 녹, 또는 알 수 없는 얼룩이 묻어 있는 상태

---

### 파일 구조 및 설명

제공되는 데이터셋 파일( `data.zip` )의 구조는 다음과 같습니다.

```
data/
│
├── train/                      # 학습 이미지 폴더 (매우 선명함)
│   ├── train_00001.jpg
│   ├── train_00002.jpg
│   └── ...
│
├── test/                       # 평가용 이미지 폴더 (블러 심함, 레이블 없음)
│   ├── test_001.jpg
│   ├── test_002.jpg
│   └── ...
│
├── train.csv                   # 학습 데이터 정답지
└── sample_submission.csv       # 제출 파일 양식 (추론 시간 기입란 포함)
```

`**train.csv` 상세**

- `Id`: 학습 이미지 파일명 (예: `train_00001.jpg`)
- `Target`: 해당 이미지의 클래스 레이블 (0 ~ 5)

**주의사항 🚨**

본 대회의 Test 데이터는 캐글 노트북 환경에서 제출 버튼을 누를 때만 접근할 수 있는 **Hidden Dataset** 형태로 평가됩니다. 여러분이 다운로드 받는 파일에는 샘플 Test 데이터만 포함되어 있습니다. 실제 채점은 약 10,000장의 블러 처리된 평가 데이터로 이루어집니다.
